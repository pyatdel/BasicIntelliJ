2. 데이터 객체의 이해

DTO (Data Transfer Object)

Data, Transfer, Object의 약자
데이터 전달이 목적
데이터 변경 불가
getter 메서드만 사용


VO (Value Object)

value, Object의 약자
값 자체를 표현
데이터 변경 가능
getter, setter 모두 사용
최근 DTO보다 많이 사용되는 추세


공통 특징

프로퍼티(멤버변수) 포함
기본 생성자 필요
